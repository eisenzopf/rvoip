# rvoip 工业级呼叫中心平台 — 产品需求文档 (PRD)

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 构建一个对标 Genesys / Five9 / Asterisk + FreePBX 的工业级 SIP 呼叫中心管理平台，覆盖从组织架构到通话质检的完整业务闭环。

**Architecture:** 基于 rvoip 纯 Rust SIP/VoIP 引擎 + PostgreSQL 18 + React 前端，模块化分层设计。后端 Axum REST API + WebSocket 实时推送，前端 React + shadcn/ui + i18n 中英文。

**Tech Stack:** Rust 2024 / Axum 0.8 / SQLx + PostgreSQL 18 / React 19 / TypeScript / Vite / shadcn/ui / SIP.js / react-i18next

**角色差异:**
- `super_admin` / `admin`: 看全局数据，所有统计
- `supervisor`: 看全局数据，不能改配置
- `agent`: 看个人仪表盘 — 我的通话数、平均通话时长、今日绩效、我的队列

**交互:**
- KPI 卡片点击 → 跳转到对应详情页
- 通话行点击 → 打开通话详情 Sheet
- 坐席头像点击 → 跳转坐席详情
- 事件流自动滚动 (WebSocket 推送)
- 图表支持日/周/月切换

**数据来源:**
- `GET /api/v1/dashboard` → KPI 聚合
- `GET /api/v1/dashboard/activity` → 趋势图
- `GET /api/v1/calls` → 活跃通话
- `GET /api/v1/agents` → 坐席状态
- `GET /api/v1/queues` → 队列状态
- `WS /ws/events` → 实时事件流

#### 2.2 坐席管理

**路由:** `/agents`

**业务逻辑:**
坐席是呼叫中心的核心实体。每个坐席有：
- 系统自动生成的 ID (AGT-001)

- 最大并发通话数

### 模块 3: 通话管理

#### 3.1 IVR 语音导航 (核心新增)

**路由:** `/ivr`
**权限:** admin+

**业务逻辑:**
IVR 是来电的第一个接触点。来电 → IVR 欢迎语 → 按键选择 → 路由到队列/坐席/子菜单/外线/语音信箱。

**IVR 流程模型:**

```
来电 → [欢迎语] → [按键菜单]
                    ├─ 1: 技术支持 → [队列: support]
                    ├─ 2: 销售咨询 → [队列: sales]
                    ├─ 3: 账单查询 → [队列: billing]
                    ├─ 0: 转人工   → [坐席: 值班经理]
                    └─ 超时/错误   → [重复播放/语音信箱]
```

**数据模型:**

```sql
CREATE TABLE ivr_menus (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    description TEXT,
    welcome_message TEXT,          -- 欢迎语文本 (TTS) 或音频文件路径
    timeout_seconds INTEGER DEFAULT 10,
    max_retries INTEGER DEFAULT 3,
    timeout_action TEXT,           -- 超时动作: repeat/voicemail/queue/hangup
    invalid_action TEXT,           -- 无效按键动作
    is_root BOOLEAN DEFAULT FALSE, -- 是否为根菜单 (来电入口)
    business_hours_start TEXT,     -- 工作时间开始 "09:00"
    business_hours_end TEXT,       -- 工作时间结束 "18:00"
    business_days TEXT,            -- 工作日 "1,2,3,4,5" (周一到周五)
    after_hours_action TEXT,       -- 非工作时间动作
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE ivr_options (
    id TEXT PRIMARY KEY,
    menu_id TEXT REFERENCES ivr_menus(id) ON DELETE CASCADE,
    digit TEXT NOT NULL,           -- "1", "2", ..., "0", "*", "#"
    label TEXT NOT NULL,           -- 显示标签 "技术支持"
    action_type TEXT NOT NULL,     -- route_to_queue/route_to_agent/sub_menu/transfer/voicemail/hangup
    action_target TEXT,            -- 目标 ID (queue_id/agent_id/menu_id/外线号码)
    announcement TEXT,             -- 转接前播报 "正在为您转接技术支持..."
    position INTEGER DEFAULT 0,   -- 排序
    UNIQUE(menu_id, digit)
);
```

**页面布局:**

### 模块 11: SIP Trunk 管理 (新增)

**路由:** `/trunks`
**权限:** super_admin

```sql
CREATE TABLE sip_trunks (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    provider TEXT,                -- "运营商A", "阿里云"
    host TEXT NOT NULL,           -- SIP 服务器地址
    port INTEGER DEFAULT 5060,
    transport TEXT DEFAULT 'UDP', -- UDP/TCP/TLS
    username TEXT,
    password TEXT,                -- 加密存储
    max_channels INTEGER DEFAULT 30,
    active_channels INTEGER DEFAULT 0,
    registration_required BOOLEAN DEFAULT FALSE,
    status TEXT DEFAULT 'active',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE did_numbers (
    id TEXT PRIMARY KEY,
    number TEXT NOT NULL UNIQUE,  -- 外线号码 "021-12345678"
    trunk_id TEXT REFERENCES sip_trunks(id),
    assigned_to TEXT,             -- queue_id 或 agent_id 或 ivr_menu_id
    assigned_type TEXT,           -- "queue", "agent", "ivr"
    description TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```
